# CLI Contract

All commands are dispatched via `typer`. Help text generated by typer is
authoritative; this document gives the semantics.

Global flags available on every command:

| Flag              | Type            | Default | Notes                                     |
|-------------------|-----------------|---------|-------------------------------------------|
| `--config`        | Path            | auto    | Path to `pyproject.toml` override         |
| `--exclude`       | List[str]       | []      | Additional glob patterns to exclude       |
| `--format`        | `text|json|markdown` | `text` | Output format                        |
| `--out`           | Path            | stdout  | Destination file                          |
| `--fail-on`       | str expr        | none    | Non-zero exit when expression is truthy   |
| `--no-color`      | flag            | off     | Disable ANSI colour in text output        |
| `-v/--verbose`    | flag            | off     | Include debug detail in stderr            |

`--fail-on` accepts a tiny expression grammar: `metric OP number`, where
`metric ∈ {score, trace_depth, file_jumps, wrapper_depth, thin_wrapper_count,
flow_fragmentation, context_width, pass_through_ratio}` and
`OP ∈ {>, >=, <, <=, ==}`.

---

## `readerfriction scan <path>`

Walks `<path>` and reports metrics for every detected entrypoint plus a
project-level summary.

Output (JSON form):

```json
{
  "schema_version": "0.1",
  "root": "/abs/path",
  "scanned_files": 12,
  "parse_errors": [],
  "entrypoints": [
    {
      "ref": {"qualname": "pkg.cli.main", "file": "pkg/cli.py", "lineno": 7},
      "metrics": {"trace_depth": 4, "file_jumps": 3, "...": "..."},
      "score": 29
    }
  ],
  "summary": {"score": 29, "...": "..."}
}
```

Exit codes: `0` success · `1` `--fail-on` triggered · `2` usage · `3` error.

## `readerfriction trace <file>:<func>`

Prints the chosen trace path from `<func>` as an entrypoint.

JSON schema: `trace-result.schema.json`.

## `readerfriction explain <file>:<func>`

Per-function breakdown: wrapper classification rules, local arity, and
contribution to the trace metrics.

JSON schema: `explain-result.schema.json`.

## `readerfriction report <path>`

Same data as `scan` but renders a human report. `--format markdown --out
<file>` is the intended usage for PR comments.

## `readerfriction diff <path> --base <ref> --head <ref>`

v0.1 scope: `<ref>` is a local directory path (not a git ref). Full git-ref
support is a 0.2 follow-up. Compares scan results between two paths and
emits a delta.

JSON schema: `report-result.schema.json` with `delta: {...}` section.
